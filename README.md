# 这游戏在骗你 · M0 原型

会对你撒谎的网页解谜游戏。血条是假的、提示是反的、最华丽的门是画上去的。
完整设计见 [PRD.md](./PRD.md)。本目录是 **M0 里程碑原型**（PRD §19）。

## 运行

```bash
npm install
npm run dev        # 开发服务器 http://localhost:5173
npm run build      # tsc 类型检查 + vite 生产构建（输出 dist/，纯静态）
npm run typecheck  # 仅类型检查
npm run lint       # eslint（含"渲染层禁读 RealState"架构规则）
```

操作：方向键 / WASD 移动。**别信屏幕中央的提示。** 注意角落那行火星签名。

## 已实现（M0 + M1）

- **序章 · 教学就是陷阱**：先给你一段"蜜月期"（真教学、真奖励）建立信任，
  再用一扇华丽假门 + "按空格"陷阱背叛你。真出口朴素地藏在左上角，线索在角落签名里。
- **第一关 · 血条反相关（L1）**：血条反向显示（扣血→进度条上涨），砖块标签被掉包
  （"伤害砖"伪装成 ❤回血）。真规则是**主动求死**——把真实血量耗到 0 才通关，
  触发葬礼演出「卒」。这是本作的杀手级时刻（PRD §1.2）。
- **第二关 · 真钥匙飘在眼前（L2）**：醒目提示叫你"按 [空格] 開門"（空格只扣血），
  真正的钥匙 `[ J ]` 写在一条飘过的青色非主流签名里，夹在伤感装饰签名之间。
- **第三关 · 规则在漂移（L3）**：真实通关目标每 ~9 秒在三块彩色踏板间漂移，
  当前目标写在角落火星文里；漂移前诚实灯闪烁预警（PRD §7.4），站稳目标 ~2.4 秒通关。
- **第四关 · 伪 OS 假报错（L4）**：满屏假"致命错误"，醒目【確定】是陷阱（连点 5 次会被嘲讽），
  真正的出口藏在错误堆栈里一行低对比火星文上（PRD §15.1 伪 OS 边界 + §8.3 反循环）。
- **第五关 · 综合 + 拒绝假结算（L5）**：四层谎言一起上（血条反相关 + 假指令 + 真钥匙 J + 空格陷阱）。
  走到金色"终点"弹出假胜利「🎉 恭喜通關」——醒目的【領取勝利】是陷阱，**真正的通关是拒绝它**
  （点角落"拒絕這個結局"）。是 L4 假报错的情感镜像：它给你最想要的胜利，你要学会拒绝（PRD §5）。
- **真结局**：界面停止撒谎，从满屏葬爱故障态切到极简素颜白底
  （风格落差本身就是 truthSignal，PRD §13），把游戏内的谎言映射到现实里的 UI 操纵。

> 关卡链：序章 → L1 → L2 → L3 → L4(伪OS) → L5(拒绝假结算) → 真结局。
> 操作：方向键/WASD 移动、`J`/Enter 确认（真钥匙）、空格通常是陷阱。

### 爽感外显（PRD §16）
- **顶部章节进度点**（序章 ●○○○○○ → 第五關 ●●●●●●），随时看到推进。
- **每关结算即时战绩**：`一次識破！沒上當 ✦` / `本關被騙 N 次` + `· X 秒`（识破耗时）。
- **真结局可分享战绩卡**：称号（先知/清醒者/半信半疑/乖﹏聽話の人）+ 识破层数 / 被骗次数 / 用时
  + **双截图梗**「你以為の血條 ████ ≠ 真實 ▏」+ 可复读口号
  + 一键「複製戰績去曬」（clipboard→execCommand→可选文本三层兜底）。

### 打磨系统（M2）
- **音效双套（程序化 WebAudio，无素材）**：假事件用甜腻三角波，真事件（truthSignal/坦白）用干净正弦；首个手势解锁。
- **识破埋点**（PRD §17）：记录每关进入/通关时间、被骗次数、识破耗时；`window.__telemetry.summary()`（DEV）。
- **二周目真相视角**（PRD §7.8）：通关后解锁「真相視角重玩」，旁路所有谎言、显示真实数值、标出真目标，
  并埋一句反套娃「這次我沒騙你（敢信嗎?）」。
- **假设置面板**（PRD §5）：右上 ⚙ 打开——靜音是真的，音量滑块是假的（拖它只会被嘲讽），語言下拉也是假的。

## 架构 → PRD 对照

| 模块 | 职责 | PRD |
|---|---|---|
| `src/engine/prng.ts` | 种子化 PRNG，确定性来源 | §11.3 |
| `src/engine/types.ts` | **RealState**（真实状态，渲染层不可见）+ WinCondition 等 | §6 §11 |
| `src/engine/deception.ts` | Deception 谎言模块接口 + HintLevel 提示阶梯 | §9 §11.1 |
| `src/engine/displayLayer.ts` | **唯一渲染数据源** RealState→DisplayState（诚实投影 + 逐个谎言变换 + 提示阶梯）| §15.6 |
| `src/engine/input.ts` | 输入管线：`event.code` → remap 链 → GameAction | §11.4 |
| `src/engine/game.ts` | 引擎编排：世界步进、胜负判定、旁白、失败重生、音效触发 | §5 §8 |
| `src/engine/audio.ts` | 程序化音效（真/假两套 WebAudio）| §8 |
| `src/engine/telemetry.ts` | 识破埋点（进入/通关/耗时/被骗次数）| §17 |
| `src/render/*` | 渲染层，**只能 import DisplayState**（eslint `no-restricted-imports` 强制）| §15.6 §15.2 |
| `src/levels/*` | 关卡内容（声明式：build + deceptions + hintLadder + 旁白）| §18 |
| `src/meta/*` | 元层 DOM 场景：伪 OS 假报错 / 假胜利结算 / 假设置面板 | §5 §15.1 |

> **核心纪律已落成机制**：`src/render/**` 若 import `engine/realState`/`engine/game` 会被
> eslint 报错拦截（`npm run lint`），而不是靠人肉 code review。
