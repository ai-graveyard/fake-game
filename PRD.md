# 《这游戏在骗你》产品需求文档（PRD）

> 一句话：这是一个**仪表盘会对你撒谎**的网页解谜游戏。血条是假的、提示是反的、通关条件偷偷被改写。真正的玩法是——分辨什么可信、反向操作、破解游戏隐藏的真实规则。

- **文档版本**：v0.2（经 6 视角专家审稿后大幅扩充）
- **日期**：2026-06-27
- **类型**：元游戏（meta）认知解谜 / Web 单机
- **核心标签**：不可靠叙述、UI 欺骗、反直觉操作、aha moment

> **v0.2 变更摘要**：补齐了 v0.1 缺失的关键层——操作与物理基准（§6）、施骗者人格与情感弧线（§4）、提示升级阶梯（§9）、失败与防卡死（§8）、公平欺骗契约（§10）、引擎确定性与可测性（§11）、元层 UI 技术边界与生命周期（§15）、内容管线/排期/Playtest/裁剪/提交清单（§18–§23）。并把"葬爱美学"从皮肤升级为机制载体，确立差异化定位"不可信仪表盘"与一个杀手级时刻。

---

## 1. 愿景与定位

### 1.1 一句话定位：不可信的仪表盘（Unreliable HUD）
一款"**界面本身就是谜题**"的网页游戏：你以为在玩游戏，其实你在调查"这个游戏到底在骗你什么"。

**差异化核心**：大多数 meta 游戏骗的是**叙事 / 第四面墙**（被骗一次后恍然大悟）；本作骗的是**玩家赖以决策的、可量化的游戏内信息本身**——血条、计时器、分数、进度条这些 HUD——并要求玩家**持续、实时地反向解读**。把"仪表盘不可信"做成一场贯穿全程的对抗，是本作最尖、最原创的支点。

### 1.2 杀手级时刻（Signature Moment）
评委只会记住一个可被复述的画面。本作的记忆锚点定为：

> **「你必须主动求死，才能通关。」**
> 血条空了、屏幕弹出火星文葬礼挽联 `R.I.P 乖﹏聽話の人`＋哀乐＋大字「卒」——但你其实没死。在这个世界里，**血条越满越接近真正的死亡，主动赴死才是通关**。

要求所有关卡都服务于强化这一时刻，而非平行铺四个机制。"向游戏求死 / 拒绝活着"是本作的核心动词，葬爱美学由此从皮肤升级为机制本体（详见 §3.1、§13）。

### 1.3 设计目标
- **主打"最佳创意"**：把"不可靠界面"做到本体论层面——玩家连"什么是真的"都不确定。
- **每个发现都是 aha moment**：爽点不是"操作变强"，而是"识破谎言"的瞬间。
- **抽象但不劝退**：抽象在概念层，操作层极简（方向、确认、少数几个键），降低上手门槛。
- **骗得优雅、识破得爽**：每处谎言被识破后让玩家会心一笑，而非恼怒（硬约束见 §10 公平契约）。

### 1.4 与标杆的差异化（防"meta 缝合怪"）
目标是创意奖，必须正面回答"凭什么不是别人的模仿"。

| 标杆 | 它骗你什么 | 本作的差异锚点 |
|---|---|---|
| The Stanley Parable | 叙事 / 自由意志，旁白叙述者 | 我们骗**可量化的 HUD 数值**，且需玩家**持续反向解读**而非听旁白 |
| Pony Island / Inscryption | 第四面墙 / 伪装的游戏程序 | 同上；且**美学（葬爱/火星文）即对手人格**，非英文极简/复古 DOS 风 |
| There Is No Game | "游戏本身"的存在 | 我们不否定游戏存在，而是让你**怀疑游戏给你的每一个读数** |
| DDLC | 文件 / 角色自我意识 | 我们的恐怖来自"工具在对你说谎"，对应**算法时代被 UI 操纵**的真实焦虑 |

> **一句话防御**：如果评委说"这是 There Is No Game 的中文版"，回答——"TINGame 骗你关于'游戏存在与否'，**我们骗你关于'你眼前每一个数字'，而且骗一整局**。"

### 1.5 不做什么（防止范围膨胀）
- 不做联机、不做存档云同步、不做账号系统。
- 不做复杂操作（无连招、无精确平台跳跃要求——见 §6 操作基准，统一去掉"跳跃物理"）。
- 不做长流程 RPG，整体时长控制在 **20–40 分钟一周目**。

## 3. 核心创意：四层谎言体系

游戏的所有欺骗都归到四类机制。关卡 = 这四类的不同组合 + 升级。

### 3.1 设计原则：美学与机制同构
葬爱家族文化的内核是**用浮夸符号掩盖真实情绪、表演性的自毁**——这本身就是"表象与真实背离"，与"游戏在骗你"同构。据此立一条贯穿设计律：

> **越浮夸、越华丽闪烁、越矫情自毁的非主流表达 = 越在掩盖真实 / 越接近真线索。**
> 真线索永远藏在最伤痛矫情的签名档、QQ 空间式留言、火星文挽歌里；UI 越花哨越在骗你；**真界面是卸了妆的素颜极简**。

这让美学与机制互为隐喻，别人抄不走。宣传语可写：「一个用葬爱家族的方式骗你、也用葬爱家族的方式对你说真话的游戏。」

### L1 · 显示谎言（UI 在说谎）
界面显示的数值/状态与真实状态不符。
- 血条满 ≠ 真的满（真实血量与显示**反相关**——血条越满越接近死亡，呼应 §1.2）。
- 计时器在倒数，其实在正数 / 根本不影响结局。
- 分数越高越接近失败。

### L2 · 指令谎言（提示是反的 / 错的）
游戏明确教你的操作是陷阱。
- "按空格跳跃" → 空格其实扣血/触发死亡。
- "收集金币得分" → 金币是扣分项。
- 醒目的"开始/继续"按钮是错误选项，真正入口在不起眼处。

### L3 · 规则漂移（通关条件偷偷改写）
真实规则会随时间/操作被悄悄改动，线索藏在玩家不会注意的角落。
- 通关条件用**火星文 / 镜像文字 / 极低对比度**写在角落、版权行、加载提示里。
- 规则每隔 N 个**逻辑 tick**（非墙钟时间，见 §11.3）微调，并配预警机制（见 §7 第3关）。
- 真正的按键写在一条**飘过的非主流签名**里。

### L4 · 元层谎言（游戏本体在骗你）
打破第四面墙的欺骗，针对"游戏这个容器"本身。
- 假的"加载失败 / 报错弹窗"，其实是谜题入口（技术边界见 §15.1）。
- 假的设置项（调音量其实改难度；语言切换其实切换真假规则显示）。
- 假的"通关结算"——其实没通关，真正的结局要拒绝这个结算。

## 4. 叙事者与对手人格（谁在骗你、为什么骗）

> 这是 v0.1 缺失的灵魂。四层谎言若没有一个有动机、有人格的"施骗主体"，玩家只会把谎言当"关卡难度"，产生不了"被背叛 / 想反将一军"的张力。**此项从待决策升级为核心设定。**

### 4.1 施骗者设定
- **身份**：施骗者 = "这个游戏 / 界面"本身，自称"我"。形象是一个 **2000 年代非主流网吧界面成精的"它"**——自卑、爱表演、用火星文和闪图掩饰，渴望被看见。
- **动机（关键）**：不是恶意，而是**"怕你一眼看穿就关掉它，所以必须骗你留下来"**。这条动机一举解释了：为什么 L1→L4 层层加码；为什么真结局它会坦白；为什么它越到后面越露怯。
- **承载方式**：角落文字 / glitch 字幕 / 火星文签名，**无需配音**。每关 3–5 句对玩家**具体操作**的即时反应（你死了它幸灾乐祸、你识破了它嘴硬）。

### 4.2 语气与情感弧线
施骗者的语气随玩家进展演变，玩家的情绪也被设计成一条曲线（机制服务于情绪，而非反过来）：

| 阶段 | 施骗者语气 | 目标玩家情绪 |
|---|---|---|
| 序章（蜜月期）| 谄媚讨好、真诚教学 | 信任、放松 |
| 第1关 L1 | 得意（第一次骗成了）| 第一次被骗的错愕 |
| 第2关 L2 | 嘴硬、恼羞 | 警惕："我不再信任何 UI" |
| 第3关 L3 | 慌乱 | 掌控感："我开始预判它" |
| 第4关 L4 | 露怯 | 好奇："它为什么这么拼" |
| 第5关 + 真结局 | 卸下伪装、坦白 | 理解与抉择、释然 |

**关卡排序校验**：若某关只升机制不推情绪，要么改设计、要么调位置。

## 5. 核心玩法循环

```
观察界面  →  怀疑(哪些信息可信?)  →  实验(小代价试探)  →
识破谎言(aha!)  →  反向/真实操作  →  推进 + 暴露下一层谎言
```

- **怀疑成本要低，但试错不能无代价**：死亡可快速重来，但需有"软成本"防止无脑穷举（见 §8.2）。
- **实验必须能产出可信信息**：每次有意义的输入，至少有一条 truthSignal 级别的**可观察反馈**让玩家做归因（如"按这个键时那盏诚实灯变暗了 → 这个键有害"）。否则玩家按了键却因 UI 撒谎不知真实结果，识破只能靠运气。
- **识破之后仍要有操作（post-reveal challenge）**：识破谎言只是"开门"，门后必须有一段需要**运用该认知**的真实操作（硬约束：**每关识破后仍需 ≥20 秒有意义操作，不得是一次性开关**）。否则关卡一旦被看穿就索然无味（这是本品类最大陷阱）。
- **难度曲线**：前期教"不要信 UI"→中期教"线索藏在哪"→后期"怀疑错对象 / 真话伪装成谎言"，而非简单叠层数。

## 6. 操作与物理基准（MVP）

> v0.1 完全没定义"角色怎么动"，而序章/L2 又写了"跳跃键"，自相矛盾。此节统一规格。

- **操作范式**：**固定单屏 2D 俯视**，方向键 8 向移动 + 一个确认/交互键 + 少量功能键。**取消跳跃物理**（规避平台跳跃精度，符合 §1.5），原文案中的"跳跃键"统一改为"**前进键 / 穿门键**"。
- **镜头**：固定单屏，不做滚动（避免"镜头也撒谎"的复杂度）。所有线索都在同一屏内（含角落），需保证不同 dpr/缩放下不被裁切（§15.3）。
- **碰撞**：格子或 AABB 二选一（推荐格子，便于确定性回放）。可交互物用高亮 / 范围触发。
- **RealState 字段补充**：在 `hp / position / score / winCondition / activeKeys` 基础上补 `velocity / facing / collisionMap`。
- **输入语义三分**：玩家**看到的提示键**（谎言）／玩家**实际按下的物理键**／系统**执行的动作**——数据结构上分三字段（详见 §11.4 输入管线），便于真相视角回放"你以为按了 X，其实触发了 Y"。

## 7. 关卡设计（MVP：序章 + 5 关 + 真结局）

> 原则：每关引入 1 个新谎言并复用旧机制；**且在每个机制内做"基础 + 反转"两段**，在 6 关内拉长有效深度。每关补三个设计要件：**twist 变体 / 目标情绪 / 识破后挑战**。

### 7.1 序章 · "教学就是第一个陷阱"（含前 120 秒脚本）
序章是整个游戏成败的第一道闸，必须既建立"信任"以备后续打破，又把第一个 aha 提前。

**前 120 秒逐拍脚本**：
- `0–20s` **蜜月期**：所有 UI 真实可信、教学真的有效、给一两次**毫无陷阱的正反馈**（"做得好！"并真的奖励）。建立"这游戏挺正常/友好"的信任锚点。
- `20–40s` **第一次低成本背叛**：在玩家**已放松**的时刻第一次撕破（如一次真实"存档成功"后紧接一次假存档）。第一次说谎的时机 = **信任峰值后 0.5 秒**，代价低但情感上明确"刚才那个友好的它骗了我"。
- `40–75s` **第一个 aha 落地**：引导玩家做出第一个"违抗 UI"的微操作（往反方向走一步），立刻给强烈正反馈——让他尝到"违抗 UI = 对"的甜头。
- `75–120s` **钩子**：用一句话/一个画面预告"后面还有更多层骗局"，建立追看动机。

**归因引导（防"以为是 bug / 手残"）**：冷启动玩家照做后死，最自然的归因是"我手残 / 游戏有 bug"，而非"它故意骗我"。强制把归因导向"它故意"：
- "照做必死"做成 **100% 确定性**（连死 2 次排除手速问题）；
- 死法**荒诞到不像 bug**（按"前进"角色原地爆炸 + 嘲讽音效）；
- 死亡瞬间一闪而过的元层文字 `乖﹏聽話の人都死了★`，点破"这是设计"但不剧透解法。

- **目标情绪**：信任 → 第一次错愕。
- **验收**：找 5 个不知情玩家盲测，**≥4 人能在 90 秒内说出"它是故意的"**而非"卡了/我手残"；序章流失率 < 设定阈值。

### 7.2 第 1 关 · 显示谎言（L1）
- **基础**：血条与真实血量反相关（UI 越满越接近死亡），真实状态线索 = 角色颜色/抖动/背景音。
- **反转（twist）**："血条这次居然是真的"——惩罚**过度怀疑**，教玩家"不能无脑反着信，要验证"。
- **识破后挑战**：要求玩家**主动管理**血条停在某个反向区间内躲避（动态而非静态正确）。
- **目标情绪**：被骗的错愕。

### 7.3 第 2 关 · 指令谎言（L2）
- **基础**：屏幕大字滚动"正确操作指引"全是反的；真按键写在飘过的签名里 `✿﹏唯ヽ爱╮真實前進鍵→[ J ]﹏✿`。
- **反转**：真键**周期性变化**（写在不同飘过的签名里），考验持续读取而非记一个键。
- **识破后挑战**：边操作边持续读签名锁定当前真键。
- **目标情绪**：警惕、"我不再信任何 UI"。

### 7.4 第 3 关 · 规则漂移（L3，含可玩的预警机制）
> "边玩边盯角落火星文、每 10 秒突袭改写"在纸面酷、实操是认知地狱（人眼无法同时盯中心和角落）。必须把"被偷袭"改成"我注意到要变了 → 主动去读"。
- **基础**：通关条件每隔若干 tick 改写，火星文写在版权行 `© 2026 別信丄面の話 真實目標：站紅方塊上3秒`。
- **预警机制**：规则改写前，truthSignal 通道先给 **0.5–1 秒诚实预警**（那盏"从不撒谎的像素灯"闪一下）；改写瞬间游戏**短暂软暂停/降速**给读取窗口。
- **硬约束**：**改写间隔 ≥ 火星文读取时间 × 2**（呼应 §13 的"5 秒可读"红线，间隔 ≥10 秒起步，随关卡推进缩短形成内部曲线）；同一时刻**只有一个真目标**。
- **识破后挑战**：制造"两个互斥目标快速交替"逼玩家做时机判断。
- **目标情绪**：掌控、开始预判。

### 7.5 第 4 关 · 元层谎言（L4）
- **基础**：进入即弹"致命错误"假报错 / 假蓝屏（**设定为"游戏内嵌的伪 OS"而非冒充真系统**，技术边界见 §15.1）。玩家若本能"关闭报错"=失败；需与报错内容交互（堆栈里藏指令）。
- **假设置面板**：音量滑块实际调"谎言强度"。
- **反循环兜底**：连续点"关闭"5 次后，第 6 次不再判失败而触发针对性提示（报错弹窗抖动 + 堆栈某行高亮，见 §8.3）。
- **识破后挑战**：在伪 OS 里完成一段需要"把界面元素当假货操作"的流程。
- **目标情绪**：好奇"它为什么这么拼"。

### 7.6 第 5 关 · 综合 + 真结局
- **综合**：四层谎言同时存在、互相矛盾，玩家交叉验证。**验证基准 = 玩家自己的"输入→真实结果"因果链**（见 §11.5 truthSignal 悖论的解法）。
- **杀手级时刻收束**：四层叠加后，唯一出路是 §1.2 的"主动求死"。
- **拒绝假结算**：假结算页"恭喜通关"出现 → 真正通关条件是**拒绝/不接受这个结算**（不点确定、关掉它、或输入角落给出的反指令）。
- **目标情绪**：理解与抉择。

#### 真结局坦白脚本大纲（情感落点）
"界面停止撒谎 + 元层独白"不能是说明文，必须回收前文关系：
1. **动机揭示**：呼应"怕被一眼看穿就被关掉"。
2. **关系回收**：用 localStorage 记录的关键事件具体点名玩家这一路的真实选择——"第 2 关你花了 X 次才信那个签名""你那次差点接受了假结算"——让坦白显得"它一直在看着你"。
3. **位置反转**：现在轮到**它请求你相信它说的这句是真的**（制造"我还要不要信"的最后一次抉择）。
4. **主题升维（指向现实）**：独白里至少一句把游戏内谎言显式映射到玩家现实——"我骗你的这些（进度条、'仅剩 1 件'、'99 人正在看'、深色模式按钮），你每天在真界面里都信着。"让结尾从"机制揭晓"升级为"情感命题"。
- **验收（替代不可测的"有回味"）**：≥60% 测试者能复述施骗者的动机，且自报"被它认真对待 / 有点心软 / 释然"的情绪。

### 7.7 终局后的元层余韵
最强记忆点常发生在"你以为结束之后"。按风险分级，**严守伦理红线**：
- **安全档**：通关后真把 `document.title` 改成「已关闭/谢谢你看穿我」、favicon 变灰、假结算页留在历史里。
- **记忆档**：localStorage 记住"你已识破过我"，玩家再打开时它以更诚实的语气开场（"又是你"），且不再骗第一层。
- **伦理红线（明确不做）**：不弹真系统弹窗、不阻止关闭页面、不伪装成杀毒/病毒警告、不做任何真正破坏体验的事。

### 7.8 复玩与可选内容（重新定位）
- **真相视角（升级）**：不要把它做成"看穿后无聊的二周目"。两个方向二选一：
  - **(A) 真相视角本身也撒谎**——所谓"真相"其实是第五层谎言，把"连真相都不可信"的本体论主题推到底；
  - **(B) 谎言会进化**——一周目识破的套路二周目失效，游戏针对你的解法定制新骗局。
  - 定调：**核心乐趣不止初见，而在"你与一个会学习的骗子的反复博弈"。**
- **可选系统（P1/P2，不堆量）**：
  - 隐藏关/隐藏结局：奖励"过度怀疑"的玩家（连真线索都不信、走第三条路 → "骗子也被你看穿"的隐藏房间）。
  - 收集物"谎言碎片/真相碎片"：藏在角落（呼应"扫角落"训练），集齐解锁开发者元层吐槽。
  - 极速识破/盲打模式：已知答案后比拼最少试错/最短时间，把"看"变成"玩"。

## 8. 失败、重来与防卡死

> v0.1 的"死亡=零惩罚快速重来"在"UI 撒谎"的游戏里会摧毁谜题，且死亡本身就是最该被撒谎的对象。

### 8.1 死亡作为谎言素材
把"死亡"纳入 Deception 素材库：至少定义一处**假死亡**（黑屏「YOU DIED」实则进入隐藏路径）和一处**假复活/假通关**。死亡状态不是中性功能。

### 8.2 防穷举的"软成本"
零惩罚瞬间满血会让玩家无脑乱按穷举每个键，绕过"观察→怀疑→识破"。对策：
- 重来时关卡状态/线索位置**轻微变化**，或试错有**累积可见反馈**（真实血量恢复需代价），让"乱按不优于观察"。
- 量化"小代价"：单次试错可在数秒内挽回，但**盲目连续乱按会累积进入更深/更难读的谎言状态**。

### 8.3 防 soft-lock 三道兜底
几个机制天然能制造死锁（L3 目标永远在变、L4 反复点关闭、L5 推理链断裂），必须有兜底：
1. **全局滞留监测**：任一关推进度为 0 持续 180s 且提示阶梯走完最高级时，强制把该关唯一真线索通道短暂切到可读现代 UI 一帧（仍需玩家自己操作，只保证信息已送达）。
2. **反循环检测**：同一无效动作累计 5 次后，第 6 次不判失败而触发针对性提示。
3. **兜底跳关**：任一关滞留 >5 分钟，提供一个隐蔽的"我卡住了"入口（藏在角落，符合世界观），允许带"未亲手识破"标记跳过，二周目真相视角补看谜底。
- **验收**：每关用"故意不扫角落、只做最直觉操作的笨玩家"脚本跑一遍，确认 5 分钟内必有非答案性救援触达。

## 9. 提示升级阶梯（Hint Ladder）

> v0.1 的 truthSignal 只是接口字段 + 一句"隐性提示随失败增多"，无法落地。每个 Deception 模块**必填** `hintLadder` 配置。

按 **"失败次数 OR 滞留时长"双触发（取先到者）**，每级只增加信息、**永不替玩家完成"反向操作"那一步**（保住 aha）。以第 2 关（真按键藏签名）为例：

| 级别 | 触发 | 给什么 |
|---|---|---|
| L0 自然态 | — | 签名带正常飘过，不强调 |
| L1 | 失败 2 次 或 卡 30s | 承载真线索的那条签名飘过频率 ×2、停留更久 |
| L2 | 失败 4 次 或 卡 60s | 该签名出现时 truthSignal 音效轻响一次，把注意力引到角落（不给答案）|
| L3 | 失败 6 次 或 卡 120s | 签名**定格不飘走** + 关键词 `[ J ]` 短暂提高对比度到可读阈值 |
| L4 | 失败 8 次 | 旁白软提示「有些字一直在飘，但它没在骗你」|

**核心**：阶梯只缩短"发现路径"，不缩短"反向操作"。把这张表做成每个 Deception 模块的必填项，写进 §11.6 可解性校验清单。

## 10. 公平欺骗契约（设计伦理）

> 欺骗类设计最大的评审风险是被读成"廉价整蛊、不尊重玩家"。附录那句"会心一笑而非恼怒"必须落成可执行准则。每个谎言上线前过此 checklist：

1. **决策前可知**：识破所需的全部线索，必须在玩家做出错误决策**之前**就已在场（哪怕藏着）。**禁止**"只能靠先死一次才知道"的纯记忆型陷阱。
2. **可学习**：骗的规律必须确定性、可归纳。**禁止**用 `random` 制造欺骗。
3. **惩罚可逆**：惩罚"信任 UI"可以，但代价必须是"快速重来"级。**禁止**一次轻信导致长时间不可逆损失（如清空进度）。
4. **嘲讽对事不对人**：元层吐槽针对"你信了 UI 这件事"，不针对玩家智商/人格。
5. **识破后闭环**：必须给"就该这样"的反馈，让玩家觉得"是我没想到"而非"它没给我机会"。

**回看公平性（硬验收）**：任何谎言被识破后，玩家回看必须能发现真线索"此前一直公平地摆在那"（"5 秒读出关键词"是其雏形，升级为硬性验收）。
**量化情绪门槛**：盲测识破后追问"刚才是被巧妙设计了，还是被坑了？"，**被坑率 >20% 的谎言打回重做**。

## 11. 真实规则系统（核心技术抽象）

引擎核心：**显示层与真实层彻底解耦**。

```
RealState（真实，玩家不可直接见）
   │  hp, position, score, winCondition, activeKeys, velocity, facing, collisionMap...
   │
   ├──► DisplayLayer（显示层）：对 RealState 施加"谎言变换"后渲染
   │       liar(hp) = maxHp - hp        // L1 反相关
   │       liar(prompt) = fakePrompts   // L2 假指令
   │
   └──► RuleEngine（规则引擎）：真实 winCondition 可被改写（L3），由种子化 PRNG 驱动
            每个谎言是一个可插拔的 "Deception" 模块
```

### 11.1 Deception 模块接口（更新：确定性签名 + 必填提示阶梯）
```ts
interface Deception {
  id: string;
  type: 'display' | 'instruction' | 'rule' | 'meta';
  transformDisplay?(real: RealState): DisplayState;          // 改变显示
  remapInput?(key: string): GameAction | null;              // 改变按键含义
  // 禁止内部调用 Math.random/Date.now；随机性与时间从 ctx 注入
  mutateRule?(rule: WinCondition, ctx: {tick: number, rng: PRNG, inputs: Input[]}): WinCondition;
  truthSignal: TruthHint;     // 每关 ≥1 条诚实通道（必填）
  hintLadder: HintLevel[];    // §9 提示升级阶梯（必填）
}
```

### 11.2 代码漏底 → 正式设计成玩法（原待决策 1 升级为已定决策）
纯前端 SPA 的全部逻辑、RealState、winCondition、truthSignal 触发条件都会随 JS 下发到客户端，可被读取/反混淆/打断点。混淆只是拖延，挡不住本作受众（会翻代码的硬核 meta 玩家）。**对一款主题就是"欺骗与识破"的游戏，源码可读不是 bug 而是必然，应吸纳进设计：**
- bundle 里**故意埋诱饵**：可读的**假 RealState / 假 winCondition**（红鲱鱼），真逻辑用运行时计算/编码隐藏——让"读代码"也变成一层欺骗（代码也在骗你，呼应主题）。
- console 彩蛋：`window` 上挂伪 debug 函数 `game.reveal()`，返回又一层谎言或一句嘲讽，让"作弊"也被预料到。
- 真正不能泄露的极少数答案（如真结局触发指令）：拆分编码/延迟下发/时间种子派生。
- 验收目标改为：**"拆解者获得的是另一种乐趣，而非剧透"**。

### 11.3 确定性与可测试性（CI 门禁）
L3 漂移若用 `Math.random()`/`Date.now()`，则结果不可复现、无法自动测可解性、玩家 bug 无法重放、真相视角无法精确回放。**确定性契约**：
- 所有随机性来自单一可注入的**种子化 PRNG**（mulberry32/xorshift）；时间用**离散逻辑 tick**（受游戏循环驱动，后台暂停），不用墙钟。
- 关卡用固定 seed 时谎言序列**完全可复现**；存档记录 seed，bug 报告一键重放。
- **自动化可解性测试（CI 门禁）**：对每个 seed，用状态搜索把 winCondition 在每个 tick 展开，断言"存在一条满足真规则的操作序列"且"对应 truthSignal 在该时刻可见/可达"。关卡合入必须过此 solver 测试。
- **真相视角** = 用同一 seed 重放 + 旁路 DisplayLayer，直接渲染 RealState 轨迹（不必缓存全量真实数值，避免明文落盘被偷看）。

### 11.4 输入解析管线（多阶段 + 防输入法吞键）
一帧里会有多层欺骗叠加 remap，且中文玩家很可能开着输入法（IME 会吞掉 keydown 字符）。管线定为显式多阶段：

```
物理事件(用 event.code 位置码，非 event.key) → 按 type 排序的 remap 链
(display→instruction→rule→meta，定义结合律与短路规则) → 真实 GameAction → RuleEngine
```
- 一律用 `event.code`（位置码，免受布局/IME 影响）；进入游戏请求 focus，用不触发 IME 的捕获层。
- 定义多 Deception 同时 remap 同一键时的**合成优先级**（"最后一个 meta 胜出"还是"链式变换"），必须可预测且进可解性测试。
- 边界用例清单：长按、组合键、失焦粘滞、移动端触屏映射（MVP 不做但先记录）。

### 11.5 truthSignal 的可信度悖论
母题是"任何界面元素都可能骗你"，那玩家凭什么信 truthSignal？若它永远诚实就成了"只看那盏灯"的后门；若偶尔撒谎可解性就破产。解法：
- **前期（序章–L2）truthSignal 绝对诚实**，建立信任（可解性地基）。
- **在 L4/真结局做一次精心设计的"连 truthSignal 都骗了你"的高潮背叛**，且这次背叛配一条**更元的新真值通道**——玩家自己的"输入→真实结果"因果链（"我按了什么，真实发生了什么"永不改变）。
- **写明**：可解性的最终地基不是某个 UI 信号，而是"**输入-真实结果的因果链对玩家始终开放可推理**"，truthSignal 只是脚手架。

### 11.6 可解性保证（只能由外测判定）
- 每个谎言必须配 truthSignal + hintLadder，否则不公平。
- **关键纪律**：做这游戏的人知道所有真相，**永远会系统性高估线索可发现性**。因此**可解性只能由 fresh-eyes 外测判定，不可由团队自评**。内部测试降级为"只验证无崩溃/流程可走通"。
- 量化门槛：每关在新玩家中的"无提示自力识破率"≥60%，低于则加强该关 truthSignal（提对比度/增频次/加音效冗余）。

## 12. UI / UX 设计

- **视觉风格**：刻意"标准、廉价、像默认模板"的游戏 UI（让谎言更可信），后期逐渐"故障化（glitch）"。注意"廉价感"要像**正经免费小游戏**，而非"坏掉的网页"（否则被读成低质量直接劝退）。
- **关键元素**：血条、计时器、分数、提示框、按钮、版权页脚、加载提示、滚动签名/弹幕带、（假）设置面板、（假）报错弹窗。
- **角落文本通道**：版权行、加载 tip、滚动签名带——L3/L4 线索载体，需固定存在以训练玩家"扫角落"。
- **诚实暗号（truthSignal）样式库**：特定音效、角色微抖动、某个从不撒谎的像素灯（信任规则见 §11.5）。
- **无障碍（升级为"冗余线索通道"，见 §12.1）**。

### 12.1 无障碍：冗余通道而非全局开关
低对比度/火星文本身就是谜题载体，一个标准的"高对比度/读屏/放大"开关会**直接显示 L3 答案**。这是机制级冲突，不能用开关调和。原则：**任何真线索都不能只靠视觉对比度/字形承载。**
- 每条真线索强制配 **≥2 个独立感知通道**：视觉（低对比度文本）+ 非视觉（特定音效节奏 / 等价但同样需判真假的 aria-label）。让低视力玩家用声音也能玩，且不破坏明眼人的谜题。
- "辅助模式"**不显示真值**，只做"不改变谜题、只改变可读性的等价替换"（把火星文真线索换成高对比度但**语义同样模糊**的普通字——把"看不清"和"想不到"解耦）。
- 技术约束：承载谎言/线索的元素一律 `aria-hidden` 或纯绘制，**禁用语义化 `<progress>`/`aria-valuenow` 表达会撒谎的数值**（否则读屏直接剧透，见 §15.5）。
- 正常支持且不冲突的两项：键盘可达性、`prefers-reduced-motion`（glitch/闪烁需尊重）。
- 硬验收：低视力/色盲模拟下，前 3 关真线索可发现率 ≥ 普通玩家的 70%。

## 13. 视觉与文案风格：克制的"葬爱家族"风（机制载体）

整体调性借用 2008–2012 QQ 空间 / 非主流"葬爱家族"美学，**但它承担机制功能（§3.1），不是纯装饰**：越浮夸自毁 = 越在掩盖真实 / 越接近真线索。

### 用在哪（Do）
- **角落线索载体**：版权行、滚动签名带、加载 tip 用火星文/繁体写真规则，玩家初期当无意义装饰。
- **真线索签名**：如 `★彡 真實の前進鍵→[ J ] 彡★`，既符合葬爱风又是功能性线索。
- **诚实暗号反差**：真界面"坦白"时切到极简干净现代 UI——靠风格落差强化"这次没骗你"（也是天然的可分享转场，见 §16）。
- **真结局演出**：从满屏非主流故障态收束为黑底单行白字，呼应"卸下伪装"。

### 怎么把握"不夸张"（Don't）
- **不满屏**：火星文只用于角落/签名/装饰位，核心玩法 HUD 用可读字体；一屏内非主流文本占比 < 20%。
- **不堆符号**：单条签名特殊符号 ≤ 4 个（如 `✿ ╮ 殇 彡`），禁止 `★彡℡ぷ灬` 连到读不出字。
- **荧光色克制**：黑/深紫底 + 1–2 个荧光强调色（品红/青绿），不要七彩跑马灯；强调色优先留给"看起来重要但其实是假的"元素。
- **可读性红线**：承载真线索的火星文必须过"陌生玩家 5 秒读出关键词"测试，否则降级符号密度（并联动 §7.4 改写间隔）。

### 风格元素库 & 工程约束
- 符号：`✿ ❤ ╮ ╰ ☆ 彡 灬 ﹏ ㄣ ㊣ ℡`（少量取用）；字面：简繁混用、生僻字替代（別/丄/の/ㄋ）、伤感词缀（殇/泪/孤/葬/夜）；版式：`▁▂▃▄▅▆▇`、呼吸动效（节制）。
- **字体工程（P0，详见 §15.4）**：这些字符散落多个 Unicode 区块，系统字体覆盖参差，缺字会渲染成豆腐块（□）使真线索不可读。**承载真线索的字必须自带 Web 字体子集 + 进"线索字符白名单" + CI 字形校验**。
- 反例（禁用）：整段不可读火星文、彩虹渐变跑马灯、盖住玩法信息的装饰。

| 火星文/非主流字体 | **中文字体版权高发雷区**，多数免费字体禁商用。选可商用字体或**图片化处理规避授权**；自带 woff2 子集 | P0 |
| 假报错/假蓝屏/假设置皮肤 | **不复刻真实品牌的 logo/配色/商标**，做"神似但原创"的伪 OS | P0 |
| UI 组件（血条/按钮/滑块/弹窗）| 自绘，保证"廉价默认模板感"可控 | P0 |
| glitch/故障视觉 | 后期氛围，可降级为 CSS 简版 | P1 |
| 背景音乐 | 随"谎言强度"变化（可作隐性提示）| P1 |
| 真结局演出 | 界面"坦白"的视听转变 | P1 |

## 15. 技术方案

| 项 | 选择（建议） | 理由 |
|---|---|---|
| 形态 | 纯前端单页 Web 应用 | 无后端，易分发 |
| 渲染 | HTML/CSS + Canvas 混合（事件层级规范见 §15.2）| UI 用 DOM（假界面逼真），玩法用 Canvas |
| 语言/框架 | TypeScript + Vite（轻量框架或原生）| 状态解耦清晰 |
| 状态管理 | 自研 RealState/DisplayLayer 双层（架构强制隔离见 §15.6）| 核心机制需要 |
| 关卡配置 | JSON/TS 声明式（Deception 模块组合 + level JSON 模板）| 便于策划填表加关 |
| 存档 | localStorage（数据结构见 §15.7）| 轻量 |
| 部署 | 静态托管（GitHub Pages / Vercel）| 零运维 |

### 15.1 元层 UI 的实现边界与"穿帮契约"
网页跑在浏览器标签内，**无法**绘制 OS 级蓝屏（覆盖不了任务栏/地址栏/浏览器 chrome）、无法可靠拦截真 F12/Cmd-Opt-I、无法伪造原生 `alert/confirm`、无法阻止刷新。一旦玩家用"真蓝屏经验"对比就穿帮。对策：
- 所有假 UI 只能是**页面内 DOM/Canvas 全屏覆盖层**，因此主动设定为"**游戏内嵌的伪 OS**"而非冒充真系统（如蓝屏顶部留一行风格化签名暗示这是演出）。
- 显式维护**做得到清单**（全屏覆盖、伪 Windows/Mac 报错弹窗、伪进度条、Page Visibility 监听切窗、`history.pushState` 伪导航）与**做不到清单**（上述限制）。
- 每个假 UI 定义**穿帮契约**：玩家做某动作时伪装**合理退场**而非白屏（点真关闭按钮 → 进入下一层）。
- 可评估 Fullscreen API 提升沉浸，但要处理用户随时 Esc 退全屏的中断态。

### 15.2 DOM/Canvas 混合的事件层级规范
- **z-index 分层表**：玩法 Canvas / HUD DOM / 覆盖层 DOM / 元层假 UI，每层标注 `pointer-events` 是否透传；元层弹窗激活时"暂停玩法输入"还是"双通道并存"逐场景写清。
- **统一坐标换算**（含 `devicePixelRatio`，Canvas 按 dpr 放大 backing store），所有命中检测走同一转换。
- **焦点管理**：玩法用**全局 keydown 捕获**而非 Canvas focus；假设置面板的 input/slider 激活时定义键盘事件归属（"音量滑块=难度"要求边操作滑块边观察玩法）。

### 15.3 分辨率与缩放
§2 写了 ≥1280×720，需补 Canvas 在不同 dpr/缩放下的 letterbox 或自适应策略，否则"角落线索"在某些缩放下被裁掉，直接破坏 L3。

### 15.4 字体与编码工程（P0）
- 自带 **Web 字体子集**（woff2，仅打包用到的字符），确保承载真线索的每个字符在所有目标浏览器**字形一致**，禁止依赖系统字体回退。
- 建立**线索字符白名单**：真线索只从已验证覆盖的字符集里挑（装饰位可放宽）。
- **CI 字形校验**：对所有线索文本检测"是否在打包字体里有字形"（fonttools 查 cmap），缺字即构建失败。
- 文本一律 UTF-8；真线索同时提供**非字体依赖的冗余载体**（关键字做成位图/SVG 描边），防单点失效。

### 15.5 可访问性的技术约束
承载谎言/线索的元素一律 `aria-hidden` 或纯绘制（防读屏朗读泄露真规则）；禁用语义化 `<progress>`/`aria-valuenow` 表达会撒谎的数值；详见 §12.1。

### 15.6 DisplayLayer 架构强制隔离（不靠纪律靠机制）
"任何渲染不得直接读 RealState"不能靠人肉约定（工期压力下必被破坏，且这种"偶尔不骗人"的 bug 极难 QA）：
- RealState **不对渲染层导出**；渲染层只能拿 DisplayLayer 产出的 DisplayState，二者分模块/包，用 ESLint `no-restricted-imports` 禁止渲染目录 import RealState，CI 拦截。
- 类型隔离：DisplayState 与 RealState 用不同的不可互换类型（branded type / 不同字段名），误用编译期报错。
- dev 模式运行时哨兵：给 RealState 套 Proxy，记录读取调用栈，发现来自渲染层路径则抛错。
- 把"DisplayLayer 是唯一渲染数据源"写成可执行的架构测试。

### 15.7 存档数据结构与生命周期
- **进度存档**：`{schemaVersion, unlockedTruthView, currentLevel, perLevelSeeds}` —— 带 `schemaVersion` 与迁移函数。
- **回放存档（真相视角所需）**：每关 `{seed, deceptionTimeline:[{layer, mutatedAt, identifiedAt}], 关键里程碑}`，配合确定性 PRNG 重放，不存全量帧。
- 防偷看：真相视角优先"同 seed 重放 + 旁路 DisplayLayer"而非缓存真实数值落盘；对 unlock 标记轻量签名/编码（抬高"翻 storage 提前解锁"门槛）。
- 损坏降级：校验失败当新档，不崩溃。
- **浏览器生命周期契约**：用 `visibilitychange` 监听切窗，规则漂移用逻辑 tick（后台暂停，避免 t 跳变）；明确刷新语义（"刷新=回安全点"用 sessionStorage 存中间态恢复，或"刷新本身是解法"——二选一写清）；`beforeunload` 仅轻提示不假设能拦住；列出每个元层场景对"刷新/切窗/退全屏/返回"的预期行为表作 QA 用例。

### 15.8 早期技术验证（Spike）
L4 是最贵、最易翻车的部分。在 M0/M0.5 用 1–2 人日做关键效果 spike：假报错弹窗 + `beforeunload` + 全屏 + 火星文在 **Chrome/Safari/Firefox/Edge × 100%/125%/150% 缩放**下的可读性。**先证可行再排内容**。

## 16. 可传播性与 Meme 设计

> 创意奖很大程度看自来水传播，但欺骗类游戏天敌是"剧透即失效"——玩家不敢分享怕剧透朋友，传播链断裂。需设计"剧透了也不亏"的传播物。
- **双截图梗**：左图玩家看到的华丽骗局界面，右图真相视角叠加的真实数值，反差强烈——**制造好奇而非剧透答案**。
- **可复读的口号化挫败**：如"这游戏让我**主动按下删除存档**才通关"。
- **为录屏而生的转场**：真结局"界面突然停止撒谎、卸妆"的转场镜头，专门为录屏设计。
- **验收联动**：把"可分享性"列为关卡验收项之一——每个 aha 点是否有对应的可截图瞬间。

## 17. 数据与埋点（让"识破"可度量）

> "30 分钟内识破前 3 关"是核心验收，但纯前端无埋点则"识破"根本无法度量（通关 ≠ 识破，可能是蒙对的）。
- 加一个轻量、**测试期可选上报**的埋点层（localStorage 缓冲 + 可选 POST 到简单收集端点，正式版可关）。
- 埋核心事件：每关进入时间、每次失败/重来、**每次"首次正确反向操作"的时间戳与触发前失败次数**、提示阶梯每级触发、滞留时长、兜底救援是否触发、放弃/关闭节点。
- 把"识破"操作化为可测代理：**识破 = 玩家在 X 秒内连续做出 ≥N 次符合真实规则的操作（而非随机分布）**，用操作序列的非随机性区分"识破"与"蒙对"。
- 配测试期 HUD / 导出 CSV，盲测后直接看每关识破率、平均识破耗时、卡点分布，接回 §20 验收和 §9 提示阶梯调参，形成闭环。

## 18. 单关内容管线

> "策划量大"不能只用"模块化复用"搪塞，需量化工序。每关标准 7 步，每步标产物与粗估工时，得出"单关成本 ≈ X 人日"再 ×6 校验排期。

1. 真规则与 winCondition 定义 → `level-N.json`
2. 选用/组合 Deception 模块
3. 写 truthSignal（强制 1 条）+ hintLadder（§9）
4. 写角落火星文文案并自测 5 秒可读性 → 文案表条目
5. 接假/真音效与视觉 → 音效 id
6. 填关卡 JSON 参数（含 seed）
7. 内部冒烟（无崩溃/流程通）+ 外部 playtest（可解性判定，§11.6）

**目标**：做一个 level JSON 模板 + 文案表（CSV），让"加一关 = 填表"而非"改代码"。

## 19. 里程碑与排期

> v0.1 里程碑无工期/人力/截止反推。先补**人力与排期假设**，再给每阶段工期与退出条件，反推是否 deadline-safe。

**人力与排期假设（占位，需用户确认）**：团队规模与角色（示例：1 名全栈 + 兼职美术，或 2 人小队）、每周可投入工时、参赛截止日（YYYY-MM-DD）。

| 阶段 | 范围 | 粗估工期 | 退出条件 |
|---|---|---|---|
| **M0 原型 + L4 Spike** | 引擎双层 + 确定性 tick + 序章 + 第1关 + §15.8 spike | ≈ 8–12 人日 | 序章盲测 ≥4/5 人 90s 内说"它故意的" |

## 20. KPI / 成功指标与验收

把比赛评分维度（创意/完成度/可玩性/呈现，按实际赛制填）逐条映射到证据点，作为提交前自检。

- **一周目完成率** ≥ 50%（测试者不靠提示通关）。
- **每关无提示识破率**与中位识破时间（前 3 关合计 ≤30 分钟）。
- **每关 aha 触发率**（出声思考中明确"啊原来"的比例 ≥ 阈值）。
- **卡关漏斗**：记录每关进入/通过人数，定位最大流失关定向修。
- **真结局**：动机复述率 ≥60% + 正向情绪自报。
- **前 2 分钟流失率** < 30%；≥70% 玩家 2 分钟内能说出"想看看它还怎么骗我"。
- **情绪门槛**：单个谎言"被坑率" >20% 打回（§10）。

## 21. Playtest 计划

> meta 谜题每名玩家只有一次"第一次被骗"，剧透即作废。验收全靠外测，必须有计划。
- **来源与名额**：目标"生面孔"≥12 人，分 3 批各 4 人，对应 M0/M1/M2；从朋友的朋友、相关 Discord/贴吧、线下招，**避免重复消耗同一批人**。
- **fresh-eyes 纪律**：事前只告诉"一个网页解谜，20–40 分钟"，**禁止剧透"会骗你"**这一核心；配"反剧透清单"（招募/页面文案/教程里绝不能暴露的信息）。
- **单人观测协议**：录屏 + 出声思考（think-aloud）+ 不干预，记录每关"首次怀疑"与"识破"时间戳和卡点。
- **迭代频率**：每批测试后 48 小时内出问题清单、修一轮再放下一批。

## 22. 范围裁剪优先级（最小可参赛版 MVCP）

> §1.5"不做什么"是边界，不是裁剪顺序。排期吃紧时（meta 项目几乎必然）按此降级，确保砍的都不伤核心创意。

- **核心不可砍**：双层引擎 + 确定性 tick、序章 + 体现 L1/L2/L3/L4 各一次的关卡、真结局、每关 truthSignal + hintLadder。
- **降级阶梯（从先砍到后砍）**：
  1. 二周目"真相视角"重放
  2. 双套音效中"假音效"精修（先用占位）
  3. glitch 美术降级为 CSS 简版
  4. 背景音乐随谎言强度变化
  5. 5 关压成 4 关（L1/L2 合并进序章后同一关）
  6. 终局后元层余韵的"记忆档"
- 每条标注"砍掉后对评审创意分的影响"。

## 23. 提交与分发 checklist

- **交付物**（按实际赛制填，常见）：稳定可玩 URL + 离线包 zip、玩法/操作说明、标题与一句话简介、封面图 + 3–5 张截图、可选预告 GIF、团队/作者信息、LICENSE。
